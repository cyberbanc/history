# PancakeSwap Prediction — загрузчик всей истории BNBUSD

Сервис читает официальный контракт BNBUSD Prediction через `rounds(epoch)`, сохраняет каждый epoch в SQLite и позволяет скачать три CSV:

- `pancakeswap_bnb_all_rounds.csv` — все запрошенные epoch, включая отменённые/незакрытые;
- `pancakeswap_bnb_valid_rounds.csv` — только корректно закрытые UP/DOWN;
- `pancakeswap_bnb_outcomes.csv` — компактная последовательность для анализа паттернов.

## Railway

1. Создайте новый пустой GitHub-репозиторий.
2. Загрузите в корень репозитория все файлы из этого архива.
3. В Railway нажмите **New Project → Deploy from GitHub repo** и выберите репозиторий.
4. В сервисе откройте **Variables** и добавьте переменные из `.env.example`.
5. Обязательно добавьте **Volume** с mount path `/data`.
6. В **Settings → Networking** нажмите **Generate Domain**.
7. Откройте домен. При заданном `ADMIN_TOKEN` используйте адрес вида:
   `https://ВАШ-ДОМЕН/?token=ВАШ_ADMIN_TOKEN`
8. При `AUTO_START=true` загрузка начнётся автоматически. Статус обновляется каждые 5 секунд.
9. После завершения нажмите **Скачать только UP/DOWN CSV** — это основной файл для анализа паттернов.

## Важное

- По умолчанию конечный epoch определяется как `currentEpoch - 2`, то есть только гарантированно завершённые раунды.
- При перезапуске сервис продолжает с сохранённого checkpoint.
- Кнопка «Удалить и начать заново» очищает базу.
- Если RPC ограничивает batch-запросы, сервис автоматически переключается на параллельные одиночные запросы.
- Для массовой выгрузки лучше использовать личный RPC endpoint, а не общий публичный endpoint.
